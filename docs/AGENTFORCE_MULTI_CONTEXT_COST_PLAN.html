Agentforce Cost Attribution Expansion Plan

Plan to expand the calculator from Service Cloud Case cost analysis into org-wide Agentforce cost attribution across many agent types, business objects, channels, teams, and workflows.

Prepared: May 14, 2026 Scope: Core product architecture Deployment model: Salesforce-native External services: none

Executive Summary

The current calculator proves the first attribution path: Agentforce and service activity can be normalized into a cost ledger, linked to Cases, and explained in dashboards, reports, Case Explorer, and Agentforce Q&A.

The next version should treat Case as one supported work target, not the product boundary. Agentforce is used for sales, service, commerce, field service, internal employee work, content, policy, and custom automations. The product should become an Agentforce cost attribution layer that answers: what was consumed, where it happened, who or what caused it, which business object it supported, and whether the cost is confidently allocated.

Target product model: Digital Wallet and native Agentforce usage provide the commercial consumption signal; Salesforce business records provide operational context; the cost ledger joins them into explainable cost by agent, action, conversation, work item, team, channel, business process, and time.

Why This Change Is Needed

Current Strength

Case-level attribution works for Service Cloud scenarios and is easy to demo with Case Explorer.

Keep

Current Gap

Many Agentforce agents do useful work without touching a Case. Those costs should not be forced into Case-specific screens or labels.

Gap

Required Direction

Generalize to work-level attribution while keeping Case as the first specialized explorer.

Cost Sources

Source Role How The Calculator Should Use It
Digital Wallet / consumption records Commercial source of truth for metered consumption, credits, entitlements, and remaining capacity when exposed in the org. Use for authoritative consumption totals and reconciliation. Join to operational activity where identifiers, timestamps, agent names, or session references allow matching.
Agentforce conversation/session records Runtime evidence of agent sessions and customer or employee interactions. Create usage anchors by agent, session, channel, date, and participant. Link to work items where possible.
Agent actions and event logs Evidence of actions, tools, prompts, and automation work performed by agents. Attribute action-heavy costs to agent, action name, topic, business process, and related record.
Service Cloud records Operational context for Case, Account, Contact, queues, status, and outcomes. Continue supporting Case Explorer and service cost reporting.
Sales, commerce, field service, and custom objects Operational context for non-service Agentforce work. Link usage to Opportunity, Lead, Order, Work Order, custom work records, or configured business objects.
Org configuration and rate tables Contract rates, fallback rates, billing model, allocation rules, and confidence thresholds. Calculate estimated or actual allocated cost and flag missing-rate or low-confidence rows.

Target Data Model

Keep the existing cost ledger, but add a generic attribution layer so each ledger row can identify the business object, owner, process, and confidence level without requiring a Case.

Object / Concept Purpose Key Fields
Cost Ledger Normalized row-level cost record. Cost, credits, conversations, source, agent, action, topic, channel, outcome, timestamp, calculation basis.
Attribution Target Generic business record link for any Salesforce object. Target object API name, target record Id, target display name, target type, confidence, allocation method.
Agent Catalog Known Agentforce agents and their business ownership. Agent Id, name, version, type, owning team, business process, default allocation rules.
Allocation Rule Declarative rules that map usage to business context. Source object, matching fields, priority, target object, fallback behavior, expiration.
Reconciliation Run Compares commercial consumption against attributed ledger totals. Period, Digital Wallet total, ledger total, variance, unmatched amount, status.

Agentforce Q&A Expansion

The current Q&A action should evolve from cost summaries and Case cost into attribution questions that mirror the expanded ledger.

Question Required Data
Which agents cost the most this month? Ledger grouped by agent and date period.
Which actions are driving cost? Ledger grouped by action name and agent.
How much Agentforce cost is not tied to a business record? Attribution target null or low-confidence rows.
What did sales agents cost by opportunity? Opportunity attribution targets and agent catalog.
Where does Digital Wallet usage not reconcile to attributed ledger cost? Reconciliation run, wallet total, ledger total, variance.
What did this agent do, and why did it cost that amount? Agent, action, session, target record, and calculation basis.

Implementation Phases

Phase Outcome Deliverables
Phase 1: Generalize Ledger Attribution Ledger rows can point to any work target, not only Case. Attribution fields/object, source sync mapping, report service updates, tests.
Phase 2: Agent and Action Reporting Admins can see cost by agent, action, topic, channel, and business process. Agent catalog, action summaries, dashboard cards, grouped report options.
Phase 3: Usage Explorer Users can inspect any attributed or unallocated usage row. New LWC, controller, filters, detail panels, links to target records.
Phase 4: Digital Wallet Reconciliation Commercial consumption can be compared to attributed ledger cost. Discovery, staging, reconciliation run object, variance dashboard, health checks.
Phase 5: Configurable Target Mapping Customers can map custom agent work to custom objects. Admin rule UI, mapping metadata, validation, audit logging.
Phase 6: Expanded Agent Q&A Agentforce answers broader attribution and reconciliation questions. Q&A parser/controller updates, new tests, Enhanced Web Chat smoke tests.

Acceptance Criteria

  • Case Explorer still works for Service Cloud cases.
  • Usage Explorer can show costs not linked to Cases.
  • Dashboard separates attributed, unallocated, and low-confidence cost.
  • Reports group by agent, action, target object, business process, channel, outcome, and period.
  • Digital Wallet reconciliation shows period total, ledger total, variance, and unmatched usage.
  • Agentforce Q&A answers supported cost questions without inventing values.
  • All new labels remain customer-friendly and avoid system-value formatting.
  • All functionality remains fully Salesforce-native for this release track.

Risks And Open Questions

Risk Mitigation
Digital Wallet data availability varies by org, edition, and permissions. Use dynamic source discovery and show reconciliation readiness instead of hard failure.
Some detailed messaging data may not be SOQL-queryable in every org. Anchor on available on-platform objects first; report gaps; keep detailed off-platform APIs as future scope unless Salesforce-native access is available.
Attribution can be ambiguous when an agent touches multiple records. Store confidence, allocation method, and raw source reference; show unallocated and low-confidence rows explicitly.
Customer-specific agents may use custom objects and custom prompts. Add configurable mapping rules and audit every mapping change.
